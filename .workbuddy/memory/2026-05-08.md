# 2026-05-08 工作记录

## 歌词显示与滚动功能重构

### 问题
1. 切换歌曲按钮没有效果
2. 歌词没有滚动显示

### 修复内容

#### 1. 切换歌曲按钮修复
- **问题原因**: `MediaSessionController` 的 `next()`/`previous()`/`playPause()` 方法在没有活跃 MediaSession 时仍然返回 `true`，导致 `RightFragment` 认为操作成功，不会执行备用方案（QQ音乐广播）
- **修复**: 修改这些方法在没有活跃 MediaSession 时返回 `false`，让调用方可以尝试备用方案

#### 2. 歌词滚动功能重构
- **新增文件**:
  - `LyricLine.kt`: 歌词数据类和解析器，支持 LRC 格式解析
  - `LyricAdapter.kt`: RecyclerView 适配器，支持逐行高亮显示
  - `item_lyric_line.xml`: 歌词行布局

- **修改文件**:
  - `RightFragment.kt`: 重写歌词显示逻辑，使用 RecyclerView 替代 TextView
  - `fragment_right.xml`: 用 RecyclerView 替换 ScrollView + TextView
  - `MediaSessionController.kt`: 添加 `getCurrentPosition()` 方法获取当前播放位置

- **功能特性**:
  - 歌词逐行解析（支持 LRC 格式时间戳）
  - 根据播放进度自动高亮当前歌词行
  - 自动滚动到当前歌词行（居中显示）
  - 当前行高亮显示（颜色、字号、透明度变化）
  - 每 500ms 更新一次歌词位置

### 技术细节
- 使用 `RecyclerView` + `LinearLayoutManager` 实现歌词列表
- 歌词解析器支持 `[mm:ss.xx]` 和 `[mm:ss.xxx]` 格式
- 通过 `MediaSession.getCurrentPosition()` 获取播放进度
- 使用 `scrollToPositionWithOffset()` 实现平滑滚动到当前行

### 关于 QQ 音乐车机版歌词获取的限制

**重要发现**: QQ 音乐车机版 **不向第三方应用暴露歌词接口**，这是官方的安全限制，无法绕过。

已尝试的方式及结果：
1. **AIDL 服务绑定** - 可以绑定服务，但所有歌词相关接口返回 `code=5, error=No permission!`
2. **ContentProvider** - 需要权限 `com.tencent.qqmusiccar.common.provider`，第三方应用无法获取
3. **广播接收** - QQ 音乐车机版的广播只包含歌曲元数据（歌名、歌手、专辑等），**不包含歌词**
4. **PlayProcessProvider** - 同样受权限限制，无法访问

**当前解决方案**: 使用 **网易云音乐公开 API** 作为备用方案获取歌词
- 优点：可以获取到大部分歌曲的歌词
- 缺点：歌词可能与 QQ 音乐显示的不完全一致（时间戳可能有差异）

**诊断方法**: 调用 `QQMusicLyricController.diagnose()` 可以查看所有歌词获取途径的测试结果。
