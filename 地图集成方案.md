# 车机启动器地图集成方案

## 问题背景

你当前的中部面板（MiddleFragment）只是**启动**高德地图，而不是**嵌入显示**它。要在应用内显示地图，有以下几种方案：

---

## 方案一：高德地图 SDK 集成（推荐）

在 MiddleFragment 中直接集成高德地图 Android SDK，这是最稳定的方案。

### 步骤：

1. **申请高德地图 Key**
   - 访问 https://lbs.amap.com/
   - 注册账号，创建应用，获取 API Key

2. **添加依赖**（app/build.gradle.kts）

```kotlin
dependencies {
    // 高德地图 SDK
    implementation("com.amap.api:map2d:latest.integration")
    implementation("com.amap.api:search:latest.integration")
    implementation("com.amap.api:location:latest.integration")
}
```

3. **AndroidManifest.xml 添加权限**

```xml
<!-- 地图相关权限 -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
<uses-permission android:name="android.permission.CHANGE_WIFI_STATE" />
<uses-permission android:name="android.permission.INTERNET" />

<application>
    <!-- 高德地图 Key -->
    <meta-data
        android:name="com.amap.api.v2.apikey"
        android:value="你的高德Key" />
</application>
```

4. **修改 MiddleFragment 布局**（fragment_middle.xml）

```xml
<?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <!-- 高德地图视图 -->
    <com.amap.api.maps2d.MapView
        android:id="@+id/map_view"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />

    <!-- 顶部控制栏 -->
    <LinearLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:orientation="horizontal"
        android:background="@color/bg_card"
        android:padding="8dp">

        <Spinner
            android:id="@+id/spinner_map_mode"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content" />

        <Button
            android:id="@+id/btn_navi"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="导航" />

    </LinearLayout>

</FrameLayout>
```

5. **MiddleFragment 代码**

```kotlin
package com.sudocar.launcher.fragment

import android.os.Bundle
import android.view.LayoutInflater
import android.view.View
import android.view.ViewGroup
import androidx.fragment.app.Fragment
import com.amap.api.maps2d.AMap
import com.amap.api.maps2d.MapView
import com.amap.api.maps2d.model.LatLng
import com.amap.api.maps2d.model.MyLocationStyle
import com.sudocar.launcher.databinding.FragmentMiddleBinding

class MiddleFragment : Fragment() {

    private var _binding: FragmentMiddleBinding? = null
    private val binding get() = _binding!!

    private var mapView: MapView? = null
    private var aMap: AMap? = null

    override fun onCreateView(
        inflater: LayoutInflater,
        container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View {
        _binding = FragmentMiddleBinding.inflate(inflater, container, false)
        return binding.root
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)

        // 初始化地图
        mapView = binding.mapView
        mapView?.onCreate(savedInstanceState)
        initMap()
    }

    private fun initMap() {
        aMap = mapView?.map

        // 显示定位蓝点
        val myLocationStyle = MyLocationStyle()
        myLocationStyle.myLocationType(MyLocationStyle.LOCATION_TYPE_LOCATE)
        aMap?.myLocationStyle = myLocationStyle
        aMap?.isMyLocationEnabled = true

        // 显示交通状况
        aMap?.isTrafficEnabled = true

        // 缩放级别
        aMap?.moveCamera(CameraUpdateFactory.zoomTo(15f))
    }

    // 生命周期管理
    override fun onResume() {
        super.onResume()
        mapView?.onResume()
    }

    override fun onPause() {
        super.onPause()
        mapView?.onPause()
    }

    override fun onSaveInstanceState(outState: Bundle) {
        super.onSaveInstanceState(outState)
        mapView?.onSaveInstanceState(outState)
    }

    override fun onDestroyView() {
        super.onDestroyView()
        mapView?.onDestroy()
        _binding = null
    }
}
```

## 方案二：悬浮窗显示外部地图（画中画）

在启动器上显示一个悬浮窗，里面运行高德地图车机版。

### 优点：
- 可以使用完整的高德导航功能
- 不需要自己实现导航逻辑

### 缺点：
- 需要系统权限（SYSTEM_ALERT_WINDOW）
- 车机系统可能限制悬浮窗

### 关键代码：

```kotlin
// 检查悬浮窗权限
if (!Settings.canDrawOverlays(context)) {
    // 引导用户开启权限
    startActivity(Intent(Settings.ACTION_MANAGE_OVERLAY_PERMISSION))
}

// 创建悬浮窗显示地图
val windowManager = context.getSystemService(Context.WINDOW_SERVICE) as WindowManager
val params = WindowManager.LayoutParams(
    WindowManager.LayoutParams.MATCH_PARENT,
    WindowManager.LayoutParams.MATCH_PARENT,
    WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY,
    WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE,
    PixelFormat.TRANSLUCENT
)

// 启动高德地图到悬浮窗（需要配合 Activity 启动模式）
val intent = packageManager.getLaunchIntentForPackage("com.autonavi.amapauto")
intent?.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_LAUNCH_ADJACENT)
startActivity(intent)
```

## 方案三：多窗口/分屏模式

Android 7.0+ 支持多窗口模式，可以让你的启动器和高德地图并排显示。

### 修改 MainActivity：

```kotlin
// 在 AndroidManifest.xml 中声明支持多窗口
<activity
    android:name=".MainActivity"
    android:resizeableActivity="true"
    android:supportsPictureInPicture="true"
    ... />

// 代码中启动地图到分屏
fun launchMapInSplitScreen() {
    val intent = packageManager.getLaunchIntentForPackage("com.autonavi.amapauto")
    intent?.apply {
        addFlags(Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_LAUNCH_ADJACENT)
        // 设置分屏比例
        putExtra("android.activity.launchBounds", Rect(0, 0, width/2, height))
    }
    startActivity(intent)
}
```

## 方案四：Web 地图（最简单）

使用高德/百度/腾讯的 Web 地图 API，在 WebView 中显示。

### 优点：
- 无需申请原生 SDK Key
- 实现简单

### 缺点：
- 无法离线使用
- 功能受限（无法导航）

### 代码示例：

```kotlin
// 布局中添加 WebView
<WebView
    android:id="@+id/webview_map"
    android:layout_width="match_parent"
    android:layout_height="match_parent" />

// 加载高德静态地图
val url = "https://uri.amap.com/navigation?from=116.481488,39.990464&to=116.46534,39.999519&mode=car&policy=1"
webview_map.loadUrl(url)
```

## 推荐方案总结

| 方案 | 难度 | 功能 | 离线 | 推荐度 |
|------|------|------|------|--------|
| 高德 SDK | 中 | 完整 | 支持 | ⭐⭐⭐⭐⭐ |
| 悬浮窗 | 高 | 完整 | 支持 | ⭐⭐⭐ |
| 多窗口 | 低 | 完整 | 支持 | ⭐⭐⭐⭐ |
| Web 地图 | 低 | 有限 | 不支持 | ⭐⭐ |

**建议**：
- 如果只需要**显示地图+定位**：用方案一（高德 SDK）
- 如果需要**完整导航**：用方案三（多窗口）或保持现状（启动外部地图）
- 如果追求**简单快速**：用方案四（Web 地图）

## 关于歌词获取的补充说明

### 为什么嘟嘟桌面能获取歌词？

嘟嘟桌面可能与 QQ 音乐有**官方合作**，获得了正确的 AIDL 接口文档。或者使用了以下非官方手段：

1. **逆向工程** - 分析 QQ 音乐的 AIDL 接口
2. **辅助功能** - 读取屏幕上的歌词文本
3. **通知栏解析** - 从通知栏提取歌词

### 你的歌词获取方案

我已经为你修改了代码，采用以下策略：

1. **优先尝试 QQ 音乐 AIDL** - 遍历可能的接口参数
2. **备用网络歌词 API** - 使用网易云音乐 API
3. **MediaSession 获取歌曲信息** - 支持所有音乐 APP

如果 AIDL 方式无法工作，网络歌词 API 应该能提供大部分歌曲的歌词。

### 调试建议

1. 开启日志过滤 `QQMusicLyric`，查看 AIDL 尝试结果
2. 如果看到 "服务连接成功" 但没有歌词，说明接口参数不对
3. 可以考虑联系 QQ 音乐开放平台申请官方接口权限
